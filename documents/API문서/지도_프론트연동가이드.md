# 지도 프론트 연동 가이드

> 커들마켓 지도 기능 프론트엔드 연동 참고 문서

본 문서는 [지도_API.md](/Users/osejin/Desktop/cmarket_api/documents/API문서/지도_API.md) 를 기준으로, 프론트엔드에서 실제 화면을 어떻게 붙이면 되는지 빠르게 참고할 수 있도록 정리한 문서입니다.

---

## 1. 기본 화면 진입 플로우

### 권장 순서

1. 위치 권한 요청
2. 권한 허용 시 현재 위치 확보
3. 권한 거부 또는 실패 시 프론트 기본 좌표 사용
   - `latitude = 37.5666`
   - `longitude = 126.9784`
4. 기본 카테고리 `HOSPITAL` 로 장소 목록 조회
5. 지도 마커 렌더링
6. 마커 클릭 시 장소 상세 API 호출
7. 상세 패널 또는 바텀시트에 정보 표시

### 첫 진입 시 추천 요청 예시

```text
GET /api/places?category=HOSPITAL&latitude=37.5666&longitude=126.9784&radius=3&page=0&size=20
```

## 2. 지도 목록 조회 전략

### 권장 호출 기준

- 지도 첫 진입 시 1회 호출
- 지도 중심 좌표가 충분히 바뀌었을 때 재호출
- 카테고리 변경 시 재호출
- 필터 변경 시 재호출
- 확대/축소에 따라 반경(`radius`) 변경 후 재호출

### 프론트에서 관리하면 좋은 상태

- `selectedCategory`
- `centerLat`
- `centerLng`
- `radius`
- `isRecommended`
- `is24Hours`
- `isEmergencyAvailable`
- `animalTypes`
- `selectedPlaceId`
- `placeList`
- `placeDetail`
- `reviewList`

---

## 3. 카테고리별 호출 규칙

### 병원

- `category=HOSPITAL`
- 사용 가능 필터:
  - `is24Hours`
  - `isEmergencyAvailable`
  - `animalTypes`

예시:

```text
GET /api/places?category=HOSPITAL&latitude=37.4979&longitude=127.0276&radius=5&is24Hours=true&animalTypes=REPTILE
```

### 카페 / 식당 / 숙소

- `category=CAFE`
- `category=RESTAURANT`
- `category=ACCOMMODATION`
- 병원 전용 필터는 보내지 않는 것을 권장
- 추천만 보고 싶으면 `isRecommended=true` 사용

예시:

```text
GET /api/places?category=CAFE&latitude=37.4979&longitude=127.0276&radius=3&isRecommended=true
```

## 4. 마커 클릭 후 상세 패널 플로우

### 권장 순서

1. 사용자가 마커 클릭
2. `placeId` 확보
3. 상세 API 호출
4. 바텀시트 또는 사이드 패널 오픈
5. 상세 데이터 표시
6. 리뷰 탭 진입 시 리뷰 목록 API 호출

### 상세 호출 예시

```text
GET /api/places/101
```

### 리뷰 호출 예시

```text
GET /api/places/101/reviews?sort=latest&page=0&size=10
```

## 5. 상세 화면 권장 UI 매핑

### 공통 필드

- `name` → 장소명
- `address` → 주소
- `phone` → 전화걸기 버튼
- `operatingHours` → 영업시간
- `imageUrl` → 대표 이미지
- `isRecommended` → 추천 배지
- `reviewSummary.reviewCount` → 리뷰 수
- `reviewSummary.averageRating` → 평점

### 병원 전용 필드

- `detail.is24Hours` → `24시` 배지
- `detail.isEmergencyAvailable` → `응급진료` 배지
- `detail.animalTypes` → 특수동물 태그

### 비병원 카테고리

- 현재 `detail = null`
- 프론트에서는 null-safe 처리 필요

## 6. 리뷰 기능 연동 플로우

### 리뷰 목록

1. 상세 패널 진입 시 바로 호출하거나
2. 리뷰 탭을 눌렀을 때 지연 호출

### 리뷰 작성

1. 로그인 여부 확인
2. 미로그인 시 로그인 유도
3. 작성 폼 입력
4. `POST /api/places/{placeId}/reviews` 호출
5. 성공 시:
   - 리뷰 목록 재조회 또는 optimistic update
   - 상세의 `reviewSummary` 재조회 권장

### 리뷰 작성 요청 예시

```json
{
  "rating": 5,
  "content": "친절하고 좋아요",
  "imageUrls": []
}
```

## 7. 추천 필터 처리

### 현재 정책

- 추천 장소는 자동 상단 고정이 아님
- `isRecommended=true` 로만 필터링
- 정렬은 거리순

### 프론트 처리 방식

- 추천 체크박스 ON:
  - `isRecommended=true` 포함하여 재호출
- 추천 체크박스 OFF:
  - `isRecommended` 파라미터 제거 후 재호출

---

## 8. 지도 반경 관리 팁

백엔드는 `radius` 를 km 단위로 받습니다.

프론트에서는 아래 정도로 단순 매핑해서 시작하는 것을 권장합니다.

- 줌 레벨 높음(가까움): `radius = 1`
- 중간 줌: `radius = 3`
- 넓은 줌: `radius = 5`
- 매우 넓은 줌: `radius = 10`

주의:

- 서버 최대 반경은 현재 `30km`
- 너무 잦은 호출을 막기 위해 debounce/throttle 적용 권장

## 9. 에러 처리 가이드

### 자주 처리해야 하는 케이스

- `400 BAD_REQUEST`
  - 병원 전용 필터를 카페/식당/숙소에 보낸 경우
  - 반경/좌표 값이 잘못된 경우
- `401 UNAUTHORIZED`
  - 리뷰 작성 시 로그인 필요
- `403 FORBIDDEN`
  - 어드민 API 권한 없음
- `404 NOT_FOUND`
  - 존재하지 않는 장소 ID

### 프론트 권장 처리

- 목록 조회 실패:
  - 토스트 또는 인라인 에러
  - 기존 마커 유지 여부는 UX 정책에 따라 결정
- 상세 조회 실패:
  - 상세 패널 닫기 또는 에러 메시지 표시
- 리뷰 작성 실패:
  - 폼 유지 + 재시도 가능 상태로 처리

## 10. 어드민 화면 연동 포인트

### 장소 목록 화면

```text
GET /api/admin/places?keyword=커들&category=CAFE&isRecommended=true&page=0&size=20
```

### 장소 등록 화면

- 카테고리 선택
- 기본 정보 입력
- 병원 선택 시 추가 필드 노출:
  - `is24Hours`
  - `isEmergencyAvailable`
  - `animalTypes`

### 추천 체크박스 토글

```text
PATCH /api/admin/places/{placeId}/recommendation
```

```json
{
  "isRecommended": true
}
```

## 11. 프론트 구현 시 권장 사항

- 목록 API 응답은 마커 렌더링 용도로만 사용하고, 상세 정보는 별도 API로 조회
- `detail` 필드는 병원 외 카테고리에서 `null` 이므로 안전하게 처리
- 리뷰 목록은 상세 진입 시마다 다시 불러오는 방식으로 시작
- 좌표 미확보 시 프론트도 서울 시청 기본 좌표를 사용하면 UX가 일관적임
- 지도 이동이 잦으므로 요청 취소(`abort`) 또는 최신 요청만 반영하는 처리 권장
